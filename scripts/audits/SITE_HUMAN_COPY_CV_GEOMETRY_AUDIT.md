# Site Human Copy, CV Selection, and Geometry Animation Audit

## What was fixed

- Removed self-explaining copy like “All Writing = all writing contests.”
- Replaced Polymythcal instructions with practical user instructions: choose a filter, open a title, check dates/rules/sign-up on the official page.
- Added a direct student safety note without a glossary box.
- Renamed Saul CV controls so they read like an action: “Choose what this CV shows.”
- Changed Saul CV areas from mutually exclusive tabs into multi-select areas. The PDF now follows one area or any combination of areas.
- Renamed the Saul “All” button to “Full CV.”
- Raised visible geometry intensity across the site and kept the Indra layer softly animated while visible.
- Added `verify-human-facing-copy.js` so confusing self-explaining copy and mutually exclusive CV tabs do not come back.

## Design principle

The website should speak to the person using it, not narrate its own internal machinery. Controls should say what they do. Explanations should help someone act.
